# Power BI Dashboard — Setup Guide

> **Important note:** A `.pbix` file is a binary file created by the Power BI
> Desktop application — it cannot be generated by writing code. This guide
> gives you the exact data and step-by-step instructions to build the
> dashboard yourself in Power BI Desktop (free download from Microsoft).
> This usually takes **30-45 minutes** the first time.

---

## Step 1: Get the data into Power BI

1. Open **Power BI Desktop**.
2. Click **Get Data → Text/CSV**.
3. Select `powerbi/churn_data_for_powerbi.csv` (already included in this project).
4. Click **Load**.

Alternative (recommended if you set up MySQL): Get Data → MySQL Database →
connect to `customer_churn_db` → table `customers`. This shows the
interviewer you can connect BI tools directly to a live database, not just CSVs.

---

## Step 2: Create calculated columns (Power Query / DAX)

In **Transform Data** (Power Query Editor), or using **New Column** in the
main view, add these:

```DAX
ChurnFlag = IF(Customers[Churn] = "Yes", 1, 0)
```

```DAX
TenureGroup =
SWITCH(
    TRUE(),
    Customers[tenure] <= 12, "0-1 Year",
    Customers[tenure] <= 24, "1-2 Years",
    Customers[tenure] <= 48, "2-4 Years",
    "4+ Years"
)
```

---

## Step 3: Create key DAX measures

Go to **Modeling → New Measure** and add each of these:

```DAX
Total Customers = COUNTROWS(Customers)
```

```DAX
Churned Customers = CALCULATE(COUNTROWS(Customers), Customers[Churn] = "Yes")
```

```DAX
Churn Rate % = DIVIDE([Churned Customers], [Total Customers], 0) * 100
```

```DAX
Total Monthly Revenue = SUM(Customers[MonthlyCharges])
```

```DAX
Revenue Lost to Churn =
CALCULATE(SUM(Customers[MonthlyCharges]), Customers[Churn] = "Yes")
```

```DAX
Average Tenure = AVERAGE(Customers[tenure])
```

---

## Step 4: Build the dashboard page layout

Create one page called **"Churn Overview"** with this layout:

### Row 1 — KPI Cards (use the "Card" visual)
- Total Customers
- Churn Rate %
- Total Monthly Revenue
- Revenue Lost to Churn
- Average Tenure

### Row 2 — Bar Charts
- **Churn Rate by Contract Type** → Clustered bar chart (Axis: Contract, Value: Churn Rate %)
- **Churn Rate by Internet Service** → Clustered bar chart (Axis: InternetService, Value: Churn Rate %)

### Row 3 — Pie / Donut Charts
- **Gender Distribution** → Pie chart (Legend: gender, Values: Total Customers)
- **Contract Distribution** → Donut chart (Legend: Contract, Values: Total Customers)

### Row 4 — Line Chart
- **Customer Count by Tenure** → Line chart (Axis: tenure, Values: Total Customers),
  split by Churn (Legend: Churn) to show churn vs tenure trend

### Row 5 — Customer Segmentation Table/Matrix
- Matrix visual: Rows = TenureGroup, Columns = Churn, Values = Total Customers

---

## Step 5: Add Slicers (Filters)

Add slicer visuals for:
- `Contract`
- `InternetService`
- `PaymentMethod`
- `gender`
- `SeniorCitizen`

These let you (or the interviewer) interactively filter every visual on the
page live during the demo — this is usually the "wow" moment in interviews.

---

## Step 6: Format for a professional look

- Use a consistent color theme: **blue for "No Churn", red for "Yes Churn"**
  (matches the Streamlit dashboard and Matplotlib charts for consistency
  across the whole project).
- Add a title text box: "Customer Churn & Retention Analytics Dashboard"
- Use **View → Themes** to apply a clean built-in theme.

---

## Step 7: Save and export

1. **File → Save As → churn_dashboard.pbix** and place it in the `powerbi/` folder.
2. Optional: **File → Export → Export to PDF** for a static copy to attach
   to your resume/portfolio email.
